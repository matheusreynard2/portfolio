SOBRE O CÓDIGO DO PRODIFY

O sistema conta com as melhores práticas de programação, seguindo padrões de Clean Code e Design Patterns, como Strategy. Segue Clean Architecture com Microsserviços - Spring Cloud e utiliza os recursos técnicos mais requisitados pelo mercado quando o assunto é Java e Spring, com Angular no frontend. Se você deseja estudar essas tecnologias, este projeto é ideal para pegar de exemplo. Se você deseja programar um sistema customizado de gestão de estoque em seu estabelecimento e não sabe por onde começar, este projeto também é ideal.

 CONCEITOS TÉCNICOS UTILIZADOS NO PROJETO

Multi-threading e assincronismo com CompletableFuture em Java, Pageable, FormData e Lobs, Lombok, RabbitMQ e WebSocket (em projeto paralelo, no mesmo repositório), Swagger, DTOs, REST API, Clean Architecture Microsserviços, Spring Security, Spring Data / JPA / Hibernate, NativeQuery / JPQL, Design Patterns (Strategy), integração com APIs externas (ViaCEP, Brasil API, Google Maps, ip.info), uso predominante de boas práticas de desenvolvimento e Clean Code. Testes unitários, testes de integração e testes de carga com JUnit 5 e Mockito.

Este sistema basicamente consiste em um gerenciamento de produtos, com fornecedores relacionados.

Na página "Gerenciar produtos > Cadastrar produtos", é possível fazer o cadastro de um produto fornecendo as informações necessárias, e relacionar um fornecedor, que por sua vez é cadastrado na página "Gerenciar fornecedores > Cadastrar fornecedores".

Na página "Lista de produtos", é possível listar todos os produtos cadastrados, consultar um por um através dos filtros disponíveis, atualizar as informações do produto e excluir um produto já existente.

Dentre as funcionalidades do sistema relacionadas a produtos estão inclusas: adicionar quantidade, relacionar um fornecedor, colocar um produto em promoção e aplicar sobre ele um desconto de 0 a 100%, calcular frete, vincular uma imagem ao produto, calcular a média de preço dos produtos já cadastrados e calcular qual o preço do produto mais caro.

Em relação aos fornecedores, é possível cadastra-los fornecendo seu CEP e buscando as informações automaticamente, depois relacionar com o CNPJ da sua empresa cujos dados também são buscados automaticamente. Também na parte de Localizar Fornecedor, você pode utilizar a integração com o Google Maps para localizar o endereço do fornecedor via CEP ou via IP público. FUNCIONALIDADES

- CRUD de produtos e fornecedores
- Relação Produto X Fornecedor
- Relação Produto X Usuário
- Relação Fornecedor X Usuário
- Calculo de quantidade X valor unitário
- Cálculo de produto cadastrado com valor mais caro
- Cálculo da média de preço entre os produtos cadastrados
- Cálculo de desconto aplicado de 0 a 100% (não aceita decimais) sobre o preço do produto
- Cálculo de frete
- Campo de buscar produtos por valor unitário, nome produto e nome fornecedor
- Upload de arquivo de imagem para relacionar com o produto
- Swagger com detalhes dos endpoints
- Spring Security - JWT (Autenticação / Token / Role USER / Criptografia de senha / Expiração de token)
- Cadastro de usuários / Login / Verificação de login já cadastrado
- Contador de acessos por IP
- Certificado SSL - Endereço HTTPS
- Totalmente responsivo
- Layout Modo Dark ON/OFF
- Integração com Google Cloud APIs para informações de localização
- Integração com Via CEP API para preenchimento automático e detalhado de endereço
- Integração com Brasil API para detecção de CNPJ e preenchimento automático e detalhado de informações da empresa
- Separação da camada de Produtos em microsserviço com Spring Cloud
- Utilização de API Gateway para rotas dos microsserviços com Spring Cloud
